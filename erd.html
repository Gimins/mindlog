도메인별 엔터티 설명

① 사용자 · 온보딩 · 포인트
엔터티역할핵심 필드
USER가입·온보딩 설정 + 포인트 상태를 함께 보관하는 중심 엔터티routine_type, wearable_type, points_balance, weekly_progress
SUBSCRIPTIONFree/Premium 구독 상태 (수익 모델)plan, expires_at
POINT_LEDGER포인트 적립·사용 거래 내역 (감사 추적)txn_type, amount, balance_after, reason
포인트 잔액(points_balance)은 USER에 두고, 모든 증감은 POINT_LEDGER에 한 줄씩 기록한다. 잔액과 거래내역을 분리해야 정산 오류·중복 적립을 추적할 수 있다(last_settled_week로 주간 중복 정산 방지).
② 콘텐츠 · AI 생성
엔터티역할핵심 필드
MEDITATION_TYPE8가지 명상 유형(M01~M08) 고정 마스터type_id, brain_effect, bg_theme
DAILY_POOL매일 새벽 AI가 생성하는 추천 풀 (7일 후 만료)state_label, expires_at, voice_url
CONTENT실제 사용·좋아요로 영구 보존된 콘텐츠 (Permanent DB)target_states, positive_state, brain_explanation
핵심 이중 구조: DAILY_POOL(매일 102개 생성, 미사용 시 7일 후 삭제) → 사용자가 실행/좋아요하면 CONTENT로 이관(promote)되어 영구 보존. target_states 배열이 진단 키워드와 콘텐츠를 잇는 추천 매칭 키다.
③ 진단 · 추천
엔터티역할핵심 필드
DIAGNOSIS5개 답변 + 생체 데이터 + 분석 결과를 담는 1회 진단 세션emotion/intensity/cause, gap_detected, primary_series, state_keywords
RECOMMENDATION한 진단이 만든 3개 추천 슬롯slot_no, slot_role(직접/이완/전환)
DIAGNOSIS 한 건이 분석 1~4(감정·강도·원인·생체 gap)를 모두 저장하고, 주 계열(primary_series) 1개를 확정한다. 이 진단이 RECOMMENDATION 3행(슬롯 1·2·3)을 생성하며 각 슬롯은 CONTENT를 가리킨다.
④ 실행 · 히스토리
엔터티역할핵심 필드
USER_HISTORY명상 실행/완료/좋아요/감정 기록 (캘린더·기록 탭의 원천)completed, liked, feel_state, start_bpm/end_bpm
USER_HISTORY는 USER · CONTENT · DIAGNOSIS 세 곳을 참조한다. liked=true 행이 "내 명상 모음"이 되고, liked_count>=3이면 추천 가중치(2배)로 피드백된다. feel_state는 마무리 화면의 감정 버튼 선택값이다.

관계(Relationship) 요약

  • USER 1 : N DIAGNOSIS — 사용자는 매일 여러 번 진단
  • DIAGNOSIS 1 : N RECOMMENDATION — 진단 1건 → 추천 슬롯 3개
  • DIAGNOSIS 1 : N USER_HISTORY — 진단에서 선택·실행한 세션
  • MEDITATION_TYPE 1 : N CONTENT / DAILY_POOL — 유형이 콘텐츠·풀을 분류
  • DAILY_POOL 1 : 1 CONTENT (optional) — 사용 시 영구 콘텐츠로 이관
  • CONTENT 1 : N USER_HISTORY / RECOMMENDATION — 콘텐츠는 여러 번 추천·실행
  • USER 1 : N POINT_LEDGER / SUBSCRIPTION — 포인트 거래·구독 이력
설계 메모: 진단 답변(q1~q5)을 DIAGNOSIS에 컬럼으로 두었으나, 질문 수가 늘면 DIAGNOSIS_ANSWER 자식 테이블로 분리하는 게 유연하다. 생체 데이터도 시계열로 쌓으려면 별도 BIOMETRIC_LOG를 둘 수 있다(MVP는 진단 시점 스냅샷만 보관).
