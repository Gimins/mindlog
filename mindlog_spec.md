## 1. 왜 이 서비스인가 (Why Now)

### 사회적 필요

운동앱·식단앱이 "병원 대체"가 아닌 "일상 관리"로 자리잡았듯, 정신건강에도 같은 도구가 필요한 시점이 됐다.

보건복지부 국립정신건강센터(2024)에 의하면 국민 73.6%가 정신건강 문제를 경험했고, 2022년 대비 2년 만에 10%p 급증했다. 정신과 수진자도 2015년 289만 명에서 2021년 411만 명으로 6년간 42% 늘었다.

그러나 문제를 겪는 대부분은 임상적 치료가 필요한 상태가 아니다. 스트레스·번아웃·수면 문제처럼 일상에서 관리하면 충분한 수준인데, 자기 상태를 인식하지 못하고 방치하다 악화된다. 문제를 겪고도 병원을 방문하지 않은 비율은 73%에 달한다.

기존 명상 앱은 콘텐츠 목록만 제공할 뿐, 지금 내 상태를 진단해 맞는 것을 추천해주지 않는다. 이 공백이 명상 추천 시스템이 필요한 이유다.

효과적인 추천 시스템이 자기 상태를 인식하게 해준다면, 대부분의 마음의 문제는 예방이 가능하다. 심각한 사람에게는 병원으로 연결되는 통로가 된다. 치료의 대체가 아니라, 조기 발견의 다리다.

### 소비자 트렌드

올리브영은 2026년 핵심 트렌드로 웰니스를 선정하며, 젊은 세대의 웰니스 소비가 두 자릿수 성장세를 보이고 있다고 발표했다. 명상·마음챙김은 더 이상 중장년층의 수행이 아니다. 2026년 서울에서 열린 국제선명상대회 참가자의 55% 이상이 2030세대였다.

소비자는 이제 일괄적인 콘텐츠가 아닌 나에게 맞는 것을 기대한다. 웰니스 소비의 기준이 "좋은 콘텐츠"에서 "나한테 맞는 추천"으로 이동하고 있다.

콘텐츠는 넘치는데, 지금 내 상태에 맞는 것을 골라주는 서비스는 없다. 소비자는 준비됐다. 비어있는 건 단 하나, 지금 내 상태를 읽고 맞는 명상을 추천해주는 시스템이다.

### 명상은 비과학적이라는 오해

명상에 대한 가장 큰 진입장벽은 비용도, 시간도 아니다. 과학적이지 않다는 오해다.

그러나 현대 뇌과학은 이미 이 오해를 뒤집었다. 명상을 하면 불안과 공포를 담당하는 편도체의 활동이 감소한다. 기억과 학습을 담당하는 해마의 회백질이 증가한다. 교감신경이 억제되고 부교감신경이 활성화되면서 심박수와 혈압이 안정된다. 1979년 존 카밧진이 개발한 MBSR(마음챙김 기반 스트레스 감소) 프로그램은 현재 전 세계 병원 임상 현장에서 사용되며, 분당서울대병원·강북삼성병원 정신과도 공식 추천한다.

문제는 이 과학적 근거가 사용자에게 전달되지 않는다는 것이다. 뇌과학 기반 추천 시스템은 이 문제를 정면으로 해결한다. 막연한 권유가 아니라, 근거 있는 처방이다. 신뢰는 경험 이전에 설명에서 시작된다.

## 3. STP 전략

### Segmentation — 시장 세분화

| 세그먼트 | 핵심 감정 | 이탈 이유 | 원하는 것 |
|---|---|---|---|
| 번아웃 직장인·학생 (20~40대) | 머리 아픔·속 답답함 | 어디서 시작할지 모름 | 진단 + 루틴 |
| 명상 입문자 | 시작하고 싶지만 막막함 | 콘텐츠 과부하·반복 | 쉬운 시작 + 맞춤 |
| 웰니스 관심층 | 더 잘 관리하고 싶음 | 개인화 없음·지루함 | 히스토리·패턴 |

> 3개 세그먼트 공통 페인포인트: 모두 "어디서 시작해야 할지 모른다"에서 출발해 개인화 없이 이탈한다.

### Targeting — 목표 시장

**1차 타겟: 번아웃 직장인·학생 (20~40대)**

머리 아프고 속 답답할 때 뭔가 하고 싶은데 어떤 명상이 맞는지 모르는 사람. 이미 일기·호흡 등을 시도하지만 루틴이 안 잡힌 사람. 명상 앱을 깔고 며칠 만에 지운 경험이 있는 사람.

타겟 선택 근거:
- 국민 46.3% 심각한 스트레스 경험 (보건복지부 2024)
- 명상 앱 첫 달 이탈률 95% → 해결 가능한 구조적 문제
- 창업자 본인이 이 페르소나 (WHY TREE 세션에서 도출)
- 진단형 명상 앱 국내 전무 → 공백 시장

**2차 타겟:** 명상 입문자 (쉬운 시작 경험으로 유입) · 웰니스 관심층 (히스토리·패턴으로 락인)

### Positioning — 포지셔닝

**포지셔닝 한 문장**

> "지금 내 마음 상태와 몸의 신호를 함께 읽고, 뇌과학이 오늘 나에게 맞는 명상을 찾아드립니다."

**경쟁사 대비 차별화**

| 구분 | 기존 앱 (Calm·마보) | MIA |
|---|---|---|
| 시작 방식 | 콘텐츠 목록 → 사용자가 선택 | 상태 입력 → 앱이 추천 |
| 근거 | 이유 없는 가이드 | 뇌과학 근거 있는 처방 |
| 루틴 | 상태와 무관한 고정 콘텐츠 | 오늘 상태를 반영한 맞춤 루틴 |
| 개인화 | 가입 시 1회 설문 | 매일 실시간 진단 |

## 4. 핵심 기능 (Core Features)

### Feature 1 — 상태 진단 (핵심)

사용자가 아무 말이나 자유롭게 텍스트로 입력하면 AI가 두 단계로 분석한다.

**1단계: 감정 레이블링 (메인)**
- "지금 당신은 [분노·긴장] 상태입니다" 형태로 현재 상태를 언어로 정의
- 상태를 언어화함으로써 통제감과 안도감 제공
- WHY TREE 인사이트: 일기쓰기 효과("내 생각이 정리되고 통제감이 생긴다")를 앱이 대신 수행

**2단계: 뇌과학 설명 (부차 — 신뢰 장치)**
- "이 상태는 편도체가 과활성화된 상태입니다" 형태로 뇌과학적 근거 제시
- 사용자가 직접 요구하는 기능이 아니나, "왜 이 명상인가"에 답해 이탈 방지
- 신뢰 장치로서 지속률 향상에 기여

**입력 방식**
- 자유 텍스트 입력 ("오늘 회의에서 너무 치였어")
- 키워드 선택 (빠른 입력을 위한 감정 태그)

---

### 4-1. 1단계 상세 설계 — 자기 상태 진단

> PHQ-9 (우울 선별) · GAD-7 (불안 선별) · PSS (지각된 스트레스) · ISI (불면증) 척도 기반
> 임상 진단 도구의 측정 영역을 감성적 언어로 재설계 — 답하기 쉽고 재미있지만 뒤에서는 정확히 측정

**진단 질문 5개**

**Q1 — 신체 신호 (PHQ-15 기반)**

> 🔋 오늘 당신의 배터리는 몇 %인가요?

- 측정 영역: 신체화 증상, 번아웃 수준 → 편도체 과활성 여부
- 예시 답변: "20%도 안 될 것 같아요" / "충전이 안 되는 느낌"

**Q2 — 감정·기분 (PHQ-9 기반)**

> 🌦 지금 내 마음 날씨를 예보한다면?

- 측정 영역: 우울·감정 둔화 → 전전두엽 활성 저하 여부
- 예시 답변: "계속 흐림" / "폭풍 직전" / "맑은데 바람이 세요"

**Q3 — 불안·긴장 (GAD-7 기반)**

> 🎧 지금 머릿속에서 어떤 노래가 반복 재생되고 있나요?

- 측정 영역: 불안·반추 → 기본모드네트워크 과활성 여부
- 예시 답변: "내가 망했나 싶은 생각" / "그 사람이 왜 그랬을까"

**Q4 — 대인관계·스트레스 원인 (PSS 기반)**

> 🧩 지금 내 하루에서 가장 큰 공간을 차지하는 게 뭔가요?

- 측정 영역: 스트레스 원인 분류 (일·관계·수면·미래·신체·원인불명) → 명상 유형 선택 기준
- 예시 답변: "상사랑의 관계" / "마감" / "딱히 없는데 다 싫어요"

**Q5 — 수면·회복력 (ISI 불면증 척도 기반)**

> 🛏 어젯밤 잠자리에서 마지막으로 한 생각이 뭔가요?

- 측정 영역: 수면 질·회복력 → 자율신경계 균형 여부
- 예시 답변: "내일 회의 생각" / "그냥 아무 생각 없이 쓰러졌어요"

**진단 입력 2가지 — 주관 평가 + 생체 리듬**

> 핵심 차별화: 개인의 주관적 자기 보고만으로는 "내가 느끼는 것"만 측정한다. 여기에 실제 생체 데이터를 결합하면 "몸이 실제로 있는 상태"까지 동시에 측정한다. 이 두 가지의 교차점에서 가장 정확한 진단이 나온다.

**입력 1 — 주관 평가 (5가지 질문)**
- 사용자가 인식하는 감정·기분·스트레스 원인
- 언어로 표현된 자기 상태

**입력 2 — 생체 데이터 (웨어러블 연동)**

| 데이터 | 측정 기기 | 의미 |
|---|---|---|
| 심박수 (HR) | Apple Watch · Galaxy Watch · Fitbit 등 | 교감신경 활성 수준, 스트레스 강도 |
| 심박변이도 (HRV) | Apple Watch · Galaxy Watch | 자율신경계 균형, 회복력 수준 |
| 호흡수 (RR) | Apple Watch · Galaxy Watch | 긴장·이완 상태 |
| 수면 데이터 | Apple Health · Samsung Health | 전날 회복 수준, 수면 질 |
| 혈중 산소포화도 (SpO2) | 지원 기기 한정 | 신체 전반 회복 상태 |

> 웨어러블 미연동 시: 주관 평가만으로 진단 (기능 저하 없이 작동). 연동 시 진단 정확도 향상 안내.

**주관 평가 vs 생체 데이터 gap 감지 — 핵심 시나리오**

| 상황 | 주관 입력 | 생체 데이터 | AI 진단 |
|---|---|---|---|
| 겉으론 괜찮다고 하지만 몸은 긴장 | "그냥 좀 피곤해요" | HR 110bpm · HRV 낮음 | "몸은 지금 높은 긴장 상태입니다" |
| 스트레스를 느끼지만 실제론 안정 | "너무 불안해요" | HR 정상 · HRV 양호 | "몸은 안정적입니다. 인지적 불안 상태" |
| 주관·생체 일치 | "완전 번아웃" | HR 높음 · HRV 매우 낮음 | "신체·심리 모두 번아웃 상태" |

**AI 분석 구조**

| 단계 | 내용 |
|---|---|
| 입력 A | 5개 질문 자유 텍스트 답변 (주관 평가) |
| 입력 B | 심박수·HRV·호흡수·수면 데이터 (생체 리듬) |
| 분석 1 | 감정 분류 (기쁨·슬픔·분노·불안·무기력·혼란 등) |
| 분석 2 | 강도 측정 (경미·보통·심각 3단계) |
| 분석 3 | 원인 분류 (일·관계·수면·미래·신체·원인불명) |
| 분석 4 | 주관·생체 gap 감지 (불일치 시 생체 데이터 우선) |
| 출력 메인 | 상태 키워드 1~3개 레이블링 |
| 출력 부차 | 뇌과학 설명 (편도체·전전두엽·자율신경계) + 생체 데이터 근거 |

**상태 레이블링 키워드 50개**

6개 계열로 분류. 각 계열은 뇌과학 영역과 1:1 매핑되어 2단계 명상 추천의 근거가 됨.

**🔴 과각성·긴장 계열 (10개)**
극도의 긴장 · 통제 불능감 · 폭발 직전 · 분노 과부하 · 신경 곤두섬 · 압박감 · 조급함 · 짜증 누적 · 과민 상태 · 긴장성 두통

**🟠 불안·걱정 계열 (10개)**
만성 걱정 · 미래 불안 · 반추 중 · 최악 상상 · 결정 장애 · 불확실 공포 · 사회적 불안 · 완벽주의 압박 · 떨리는 마음 · 긴장성 불안

**🔵 무기력·우울 계열 (10개)**
방전 상태 · 감정 마비 · 무기력 · 의욕 상실 · 공허함 · 슬픔 잔재 · 자기 비판 · 고립감 · 흥미 저하 · 무감각

**🟣 혼란·과부하 계열 (10개)**
머리 과부하 · 생각 정체 · 집중력 분산 · 멀티태스킹 번아웃 · 우선순위 혼란 · 결정 피로 · 정보 과부하 · 혼돈 상태 · 방향 상실 · 생각 루프

**🟢 회복·전환 필요 계열 (10개)**
번아웃 초기 · 수면 부채 · 신체 긴장 · 회복 필요 · 자기 돌봄 부재 · 감정 소진 · 관계 피로 · 경계 필요 · 재충전 신호 · 균형 상실

**⚪ 중립·탐색 계열 (10개)**

### Feature 2 — 뇌과학 기반 명상 추천

레이블링된 상태에 따라 명상 프로그램을 랜덤 3개 추천한다.

**추천 로직 5단계**

1. 진단 결과 → 상태 계열 분류 (6개 계열 중 1개)
2. 계열에 해당하는 명상 풀 필터링
3. 풀 안에서 랜덤 3개 추출
4. 단, 같은 날 이미 한 명상은 제외 (히스토리 기반)
5. 추천 1번은 계열 최적 명상 고정, 추천 2·3번은 랜덤

**8가지 명상 유형**

| 유형 | 핵심 방식 | 뇌과학 효과 |
|---|---|---|
| 호흡 명상 | 호흡에 집중 (4-7-8, 박스 브리딩 등) | 부교감신경 활성화 → 편도체 진정 |
| 마음챙김 | 지금 이 순간 판단 없이 관찰 | 기본모드네트워크 감소 → 반추 차단 |
| 바디스캔 | 정수리→발끝 신체 감각 순차 알아차림 | 신체-마음 재통합 → 이완 반응 유도 |
| 자비명상 | 나와 타인에게 따뜻한 마음 보내기 | 감마파 증가 → 공감·긍정 정서 활성 |
| 시각화 | 평화로운 장면·공간을 마음속으로 그리기 | 전전두엽 활성화 → 코르티솔 감소 |
| 만트라 | 짧은 문구·소리 반복 | 디폴트모드네트워크 억제 → 세타파 활성 |
| 감사명상 | 오늘 감사한 것 3가지 떠올리기 | 도파민·세로토닌 분비 → 긍정 정서 강화 |
| 수면명상 | 잠들기 전 점진적 이완 (PMR, NSDR 등) | 부교감신경 우세 → 멜라토닌 분비 촉진 |

**상태 계열별 명상 풀**

| 상태 계열 | 추천 풀 | 제외 |
|---|---|---|
| 과각성·긴장 | 호흡·바디스캔·마음챙김·시각화·만트라 | 자비·감사·수면 |
| 불안·걱정 | 마음챙김·호흡·시각화·만트라·바디스캔 | 자비·감사·수면 |
| 무기력·우울 | 자비·감사·시각화·마음챙김·바디스캔 | 호흡·만트라·수면 |
| 혼란·과부하 | 만트라·호흡·마음챙김·시각화·바디스캔 | 자비·감사·수면 |
| 회복·전환 | 수면·바디스캔·자비·시각화·감사·호흡 | 만트라·마음챙김 |
| 중립·탐색 | 8가지 전체 | 없음 |

**좋아요 기능 — 개인화 피드백 루프**

- 각 명상 카드에 하트 버튼 제공
- 좋아요 누른 명상은 "내 명상 모음"에 저장 → 언제든 빠르게 재접근
- 좋아요 데이터가 추천 로직에 피드백으로 연결:
  > "분노 상태일 때 바디스캔에 좋아요 3회" → 다음 분노 상태 진단 시 바디스캔 랜덤 풀 우선 추출
- 완전 랜덤 → 나의 취향이 반영된 랜덤으로 점진적 진화
- 좋아요 누른 즉시 피드백: "[명상 이름] — 다음 [상태] 진단 시 우선 추천됩니다"

### Feature 3 — 개인 히스토리

사용자의 상태 변화·명상 경험을 기록하고 패턴을 시각화한다.

- 날짜별 감정 레이블 기록
- 패턴 분석: "지난 2주간 월요일 오전에 분노 상태가 많았습니다"
- 좋아요 기록: 계열별 선호 명상 누적 → 추천 정확도 향상
- "내 명상 모음": 좋아요 누른 명상만 모아보는 탭
- 자기 이해를 깊게 하는 경험 → 앱이 "나를 아는 존재"로 인식됨
- 락인(Lock-in) 효과: 히스토리·좋아요가 쌓일수록 이탈 비용 증가

### Feature 4 — 아침·저녁 루틴 알림

단순 알림이 아닌 상태 기반 맞춤 루틴 구조다.

- **아침**: "오늘 하루를 어떻게 시작하고 싶으세요?" 입력 → 맞춤 명상 추천
- **저녁**: "오늘 어떤 하루였나요?" 입력 → 하루 마무리 명상 추천
- 입력 → 진단 → 추천 루프가 매일 반복되며 습관 형성
- 의지 없이도 지속되는 구조 (습관은 의지가 아닌 구조로 만들어진다)

> 근거: 명상 앱 사용자가 원하는 기능 1위 — "연습을 위한 팁·알림" (JMIR Mental Health 2023)

## 5-0. 로그인 · 온보딩 화면 상세 설계

### 로그인 화면

- 소셜 로그인만 지원 (이메일+비밀번호 없음)
- 지원 소셜: 카카오 / Google / Apple
- 탭 한 번으로 가입+로그인 완료
- 약관·개인정보처리방침 동의: 가입과 함께 처리 (별도 화면 없음)
- 가입 후 → 온보딩 1로 자동 이동

### 온보딩 1 — 루틴 선택

| 선택지 | 알림 구성 | 타임라인 |
|---|---|---|
| 아침만 | 아침 알림 1회 | 아침 → CTA |
| 저녁만 | 저녁 알림 1회 | CTA → 저녁 |
| 아침·저녁 | 아침 + 저녁 2회 | 아침 → CTA → 저녁 |

- 진행 표시: 점 3개 (활성 점은 길게 늘어남)
- 뒤로가기 → 로그인 화면

### 온보딩 2 — 알림 시간 설정

| 알림 종류 | 기본값 | 설명 |
|---|---|---|
| 명상 알림 | 선택 루틴에 따라 | 아침 06:00 / 저녁 21:00 |
| 회고 알림 | 오전 8:00 | 명상 후 다음날 아침 회고 |

- time picker로 직접 설정
- 선택 루틴에 해당하는 시간만 표시

### 온보딩 3 — 웨어러블 연동

| 선택지 | 연동 방식 |
|---|---|
| Apple Watch | Apple Health API |
| Galaxy Watch | Samsung Health API |
| 기타 (Fitbit·Garmin) | HealthKit 범용 |
| 없어요, 나중에 | 질문 기반만으로 진단 |

- 연동 없어도 앱 정상 작동
- 연동 시 진단 정확도 향상 안내

### 첫 진단 안내 화면

- 진단이 무엇인지 3가지 포인트로 예고

  - 🧠 AI 뇌과학 레이블링
  - 🎵 상태 맞춤 명상 3가지 추천

- 보조: "나중에 할게요" → 홈으로 바로 이동

## 5-1. 진단 결과 화면 상세 설계 (화면 2)

### 구성 요소

**섹션 1 — 진단 상태 텍스트**

```
지금 당신은 '[상태 키워드]' 한 상태.
'[필요한 명상 방향]' 한 명상이 필요합니다.
```

- 상태 키워드: 50개 레이블 중 1~3개 조합 (예: 분노·긴장)
- 명상 방향: 키워드에 대응하는 처방 언어 (예: 머릿속을 비우고 몸의 긴장을 풀어주는)

**섹션 2 — 왜 이런 상태인가요?**

3개 층으로 구성:

| 층 | 내용 | 역할 |
|---|---|---|
| ① 사용자 대답 요약 | 입력한 텍스트를 한 문장으로 요약 | "내 말을 이해했구나" 공감 |
| ② 생체 리듬 수치 | BPM · HRV · 호흡수 3개 수치 카드 | "몸이 실제로 말하는 것" 시각화 |
| ③ 뇌과학 근거 | 생체 수치 → 뇌 상태 → 행동 변화 인과관계 | 신뢰 장치, 이탈 방지 |

뇌과학 근거 문장 구조 — 카드형 UI (문장 1개 = 카드 1개):

```
┌──────────────────────────────────────┐
│ 🧠  [편도체]가 과활성화된 신호입니다.  │  ← 뇌 이모지 + 문장 1개
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│ 🧠  [전전두엽]이 억제되어 더 예민하게  │
│     반응하게 됩니다.                   │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│ 🧠  [부교감신경]을 활성화하는 호흡     │
│     명상이 필요합니다.                 │
└──────────────────────────────────────┘
```

카드 UI 규칙:
- 문장 단위로 카드 분리 (마침표 기준)
- 각 카드 앞 🧠 이모지 포인트
- 뇌 관련 키워드 볼드 + 보라색 강조
  - 대상: 편도체, 전전두엽, 부교감신경, 교감신경, 도파민, 세로토닌, HRV, 뇌파, 알파파, 해마, 기본모드네트워크 등
- 카드 순차 페이드인 애니메이션 (0.08초 간격)
- 목적: 긴 텍스트 덩어리 가시성 개선 → 한 카드에 한 문장만

**섹션 3 — 추천 명상 3가지**

각 명상 카드 구성:

```
[아이콘] 명상 이름          [추천 N]
         시간
긍정적인 마음 상태로 · 명상 타입
```

| 추천 순위 | 선정 기준 | 명상 타입 예시 |
|---|---|---|
| 추천 1 | 상태 계열에 가장 직접 대응 | 호흡 |
| 추천 2 | 신체 이완 보완 | 휴식 |
| 추천 3 | 인지·감정 전환 | 시각화 |

**상태 계열별 긍정 변화 언어 예시**

| 상태 계열 | 추천 후 긍정 상태 |
|---|---|
| 과각성·긴장 | 차분하고 안정된 마음 · 몸의 긴장이 녹아내리는 가벼움 |
| 불안·걱정 | 지금 이 순간에 머무는 평온함 · 생각의 흐름이 느려지는 고요함 |
| 무기력·우울 | 나 자신에게 따뜻해지는 느낌 · 작은 감사가 채워지는 마음 |
| 혼란·과부하 | 머릿속이 정리되는 맑음 · 하나에 집중하는 단순함 |
| 회복·전환 | 몸과 마음이 쉬어가는 느낌 · 내일을 위한 에너지 충전 |
| 중립·탐색 | 하루를 가볍게 마무리하는 평화 · 내일을 여는 맑은 시작 |

## 5-2. 화면 2 UX — 카드 선택 방식 (방식 A)

- 3개 명상 카드가 모두 미선택 상태로 시작
- 카드 탭 → 보라색 테두리 + 우측 원형 체크 활성화 → 선택 상태
- 다른 카드 탭 → 선택 이동 (단일 선택)
- 하단 버튼: 미선택 시 "명상을 선택해주세요" (비활성) → 선택 시 "[명상 이름] 시작하기" (활성)
- 하트 버튼은 카드 선택과 독립적으로 작동 (탭해도 카드 선택 안 됨)
- 선택 후 시작 버튼 탭 → 화면 3 명상 실행으로 이동

## 5-3. 명상 실행 화면 상세 설계 (화면 3)

### 화면 구성 — 5개 영역

**영역 1 — 최상단 헤더**

```
← 뒤로가기    [명상 중]    ···
```

- 명상 유형 뱃지 (호흡 · 휴식 · 시각화 등)
- 명상 타이틀 (22px · 중앙 정렬)

**영역 2 — 기대 효과 문장**

확신의 톤으로 작성:

```
"이 명상으로 당신은 편도체의 과활성이 가라앉고,
차분하고 안정된 마음을 되찾게 됩니다."
```

- Permanent DB의 positive_state + brain_explanation 조합으로 자동 생성
- 상태 계열별 다른 문장 → AI 스크립트 생성 시 함께 생성

**영역 3 — 뇌파 시각화 + 생체 데이터**

```
┌─────────────────────────────┐
│  뇌파 (알파파)              │  ← 우상단 레이블
│  ~~~~ 웨이브 3개 애니메이션 │  ← 재생 중에만 움직임
│  델타 세타 알파 베타 감마    │  ← 하단 축 레이블
├─────────────────────────────┤
│  BPM 108  │ HRV 낮음 │ 안정 중 │  ← 실시간 수치
└─────────────────────────────┘
```

실시간 변화 로직:
- 명상 진행 → BPM 점진적 감소 (시작값 → 목표값)
- HRV 상태 변화: 낮음 → 보통 → 양호
- 뇌파 상태 변화: 안정 중 → 안정됨
- 웨이브 애니메이션: 재생 시 활성 · 일시정지 시 정지

배경 이미지 (명상 유형별):

| 유형 | 배경 컬러 테마 | 이미지 키워드 |
|---|---|---|
| 호흡 | 딥 블루 (#0C447C) | 바람, 파도, 하늘 |
| 마음챙김 | 딥 teal | 숲, 이슬, 안개 |
| 바디스캔 | 딥 퍼플 | 별빛, 우주, 빛 |
| 자비명상 | 딥 핑크 | 따뜻한 빛, 꽃 |
| 시각화 | 딥 teal | 자연, 물, 구름 |
| 만트라 | 딥 앰버 | 촛불, 모래, 사막 |
| 감사명상 | 딥 그린 | 초원, 햇살, 나무 |
| 수면명상 | 딥 네이비 | 달빛, 별, 밤하늘 |

**영역 4 — 플레이어 컨트롤**

```
[진행 시간]                [총 시간]
━━━━━━━━━━━━━━━━━━━━━  ← 프로그레스 바

   ↺15    ▶/⏸    15↻     ← 컨트롤

  ♡ 좋아요   🔊 소리   ⏱ 타이머  ← 보조 버튼
```

- 좋아요: 재생 중에도 하트 누를 수 있음 → Permanent DB에 즉시 저장
- 소리: BGM 볼륨 조절
- 타이머: 명상 종료 타이머 설정

**영역 5 — 완료 팝업 (명상 종료 시)**

재생이 끝나면 팝업 등장:

```
✦ 명상 완료

차분하고 안정된 마음이 되셨나요?
BPM이 108 → 82로 안정됐어요.

[↺ 한 번 더]  [✓ 끝내기]
```

- 시작 BPM vs 종료 BPM 비교 → 실제 효과 확인
- 한 번 더: 동일 콘텐츠 재시작
- 끝내기: Permanent DB 저장 + 히스토리 기록 → 4단계 마무리 화면으로 이동

### 데이터 저장 시점

| 이벤트 | 저장 내용 |
|---|---|
| 재생 시작 | played_at 기록 시작 |
| 좋아요 누름 | liked = true · liked_count +1 즉시 저장 |
| 한 번 더 | replayed_count +1 |
| 끝내기 | completed = true · Permanent DB 이관 확정 |
| 중도 종료 | completed = false · 히스토리에 기록 (재접근 가능) |

## 5-6. 홈 화면 상세 설계

### 화면 스타일
스토리형 — 오늘 하루의 흐름을 타임라인으로 보여줌. 가장 중요한 CTA(진단하기)를 중앙에 배치.

### 화면 구성

**섹션 1 — 상단 헤더**

```
[날짜·요일]          [히스토리] [설정]
안녕하세요

🔥 연속 N일째 명상 중          이번 주 N회
```

- 연속 명상 스트릭 → 루틴 유지 동기 강화

**섹션 2 — 나의 명상 루틴 선택**

3개 버튼으로 루틴 직접 선택:

| 버튼 | 알림 구성 |
|---|---|
| 아침만 | 아침 알림 1회 |
| 저녁만 | 저녁 알림 1회 |
| 아침·저녁 | 아침 + 저녁 알림 2회 |

시간 설정 (time picker):
- 아침 기본값: 오전 6:00
- 저녁 기본값: 오후 9:00
- 선택한 루틴에 해당하는 시간 설정만 표시
- 시간 변경 → 알림 설명 문장 실시간 업데이트
- 예: "오전 6:00에 알림이 와요. 하루를 시작하는 명상 루틴입니다."

저장 위치: 사용자 설정 DB

| 필드 | 내용 |
|---|---|
| routine_type | morning / evening / both |
| morning_alarm | 아침 알림 시간 (HH:MM) |
| evening_alarm | 저녁 알림 시간 (HH:MM) |

**섹션 3 — 오늘의 루틴 타임라인**

선택한 루틴에 따라 타임라인 구성이 바뀜:

```
아침만:
  ○ 아침 루틴 (오전 N:00 알림 예정) — 대기
  ● 지금 진단하기 (CTA) — 활성

저녁만:
  ● 지금 진단하기 (CTA) — 활성
  ○ 저녁 루틴 (오후 N:00 알림 예정) — 대기

아침·저녁:
  ○ 아침 루틴 — 대기
  ● 지금 진단하기 (CTA) — 활성
  ○ 저녁 루틴 — 대기
```

- 완료된 루틴: 초록 체크 + 카드 배경 초록
- 활성(CTA): 보라 테두리 + "지금 진단하기" 버튼
- 대기: 흐릿하게 (opacity 0.55)

**섹션 4 — 최근 명상**
- 최근 2개 명상 카드
- 명상 이름 + 날짜·시간 + 진단 상태 키워드

### 루틴별 완성 플로우

아침만 / 저녁만 / 아침·저녁 모두 동일한 플로우:

```
알림 탭 (또는 홈 "지금 진단하기" 탭)
  → 화면 1 (5가지 질문 입력)
  → 화면 2 (진단 결과 + 명상 추천)
  → 화면 3 (명상 실행)
  → 화면 4 (마무리 — 감정 상태 + 생체 요약)
  → 홈 해당 루틴 완료 표시
```

> 아침·저녁 모두 화면 4(마무리)까지 완주. 데이터 일관성 + 동일한 사용자 경험 유지.

## 7. 전략 캔버스 인사이트

MIA의 전략은 Calm과 경쟁하지 않는다. Calm이 강한 영역(수면·콘텐츠량·글로벌 브랜드)을 비워두고, Calm이 완전히 비워둔 영역(상태 진단·감정 레이블링·맞춤 추천·루틴·히스토리)을 채운다.

두 곡선이 완전히 교차하는 구조 = 블루오션 전략의 핵심 패턴.

**검증된 차별화 (사용자 리뷰 기반)**
- ✓ 쉬운 시작 경험: "어디서 시작할지 모름"이 이탈 1위 이유
- ✓ 상태 진단·맞춤 추천: 반복·비개인화가 이탈 원인
- ✓ 아침·저녁 루틴: 사용자가 원하는 기능 1위
- ✓ 히스토리·패턴: 락인 효과, 자기 이해 심화
- △ 뇌과학 근거 설명: 신뢰 장치 (사용자 직접 요구 아님)
- ★ 생체 데이터 연동: 업계 유일 — 주관 평가 + 실제 생체 리듬 교차 진단

**생체 데이터 연동 — 경쟁사 비교**

| 경쟁사 | 웨어러블 연동 수준 |
|---|---|
| Calm | Apple Health 수면 데이터 일부 연동 — 진단에 미활용 |
| 마보 | 웨어러블 연동 없음 |
| Headspace | 웨어러블 연동 없음 |
| Mindlog | HR·HRV·호흡수·수면 → 주관 평가와 교차해 진단에 직접 활용 |

> gap 감지가 핵심: 사용자가 "괜찮다"고 입력해도 HR 110bpm·HRV 낮음이면 "몸은 긴장 상태"로 진단. 스스로 몰랐던 내 상태를 앱이 잡아내는 순간이 가장 강력한 사용자 경험.

## 8. 명상 콘텐츠 DB 설계

### 명상 유형 DB (8가지)

| ID | 유형명 | 영문명 | 핵심 방식 | 권장 시간 | 뇌과학 효과 | 추천 시간대 |
|---|---|---|---|---|---|---|
| M01 | 호흡 명상 | Breathing | 호흡에 집중 (4-7-8, 박스 브리딩) | 5~10분 | 부교감신경 활성 → 편도체 진정 | 아침·낮 |

### 콘텐츠 DB 구조 — Daily Pool + Permanent DB 이중 구조

> 핵심 설계 원칙: AI가 매일 생성하는 콘텐츠는 추천 풀로 활용하고, 사용자가 실제로 경험한 콘텐츠는 영구 보존해 언제든 재접근 가능하게 한다.

**구조 흐름**

```
AI 생성 (매일 144개)
        ↓
Daily Pool (7일 유지 후 만료)
  - 매일 추천에 사용
  - 아무도 실행 안 한 콘텐츠는 7일 후 자동 삭제
        ↓ 사용자가 실행하거나 좋아요 누름
Permanent DB (영구 보존)
  - 실행한 콘텐츠 전체 저장
  - 좋아요 누른 콘텐츠 저장
  - 히스토리·좋아요 탭에서 재접근 가능
  - 재생 가능 (동일 콘텐츠 재사용)
```

**Daily Pool DB**

| 필드 | 설명 |
|---|---|
| pool_id | 풀 고유 ID |
| content_id | 콘텐츠 ID |
| state_label | 대상 상태 키워드 |
| type_id | 명상 유형 ID |
| generated_at | 생성 일시 |
| expires_at | 만료 일시 (생성 후 7일) |
| voice_url | 보이스 파일 CDN URL |
| bgm_url | 음악 파일 CDN URL |

**Permanent DB**

| 필드 | 설명 |
|---|---|
| content_id | 콘텐츠 ID (Daily Pool에서 이관) |
| type_id | 명상 유형 ID |
| title | 콘텐츠 제목 |
| script | 스크립트 텍스트 |
| voice_url | 보이스 파일 URL (영구 저장) |
| bgm_url | 음악 파일 URL (영구 저장) |
| duration_min | 소요 시간 |
| positive_state | 명상 후 긍정 상태 언어 |
| brain_explanation | 뇌과학 설명 |
| created_at | 최초 생성 일시 |

**사용자 히스토리 DB**

| 필드 | 설명 |
|---|---|
| user_id | 사용자 ID |
| content_id | Permanent DB 콘텐츠 ID |
| state_label | 진단 시 상태 키워드 |
| liked | 좋아요 여부 (boolean) |
| liked_count | 누적 좋아요 횟수 |
| played_at | 명상 실행 일시 |
| completed | 완료 여부 |
| replayed_count | 재실행 횟수 |

> 좋아요 가중치 로직: liked_count >= 3 이면 해당 상태 계열 추천 풀에서 우선 추출 가중치 부여

**사용자가 경험하는 것**

| 탭 | 내용 |
|---|---|
| 히스토리 탭 | 날짜별 실행 명상 목록 → 탭하면 동일 콘텐츠 재생 |
| 좋아요 탭 | 하트 누른 명상 모음 → 언제든 재접근·재생 |
| 추천 탭 | 오늘의 Daily Pool에서 랜덤 3개 추천 |

**스토리지 비용 관리**

- Daily Pool: 7일 만료 → 미사용 콘텐츠 자동 삭제 → 스토리지 최소화
- Permanent DB: 실제 사용된 콘텐츠만 영구 보존 → 사용자당 평균 수십~수백 개 수준
- 예상 스토리지: 사용자 1명당 ~500MB (1년 기준)

## 8-1. AI 콘텐츠 생성 파이프라인

> 핵심 차별화: Calm·마보는 사람이 직접 녹음 → 수량 한계·반복 이탈. Mindlog는 AI로 매일 자동 생성 → 무한 다양성·반복 없음.

### 생성 레이어 3개

**레이어 1 — 스크립트 (LLM)**

| 항목 | 내용 |
|---|---|
| 엔진 | Claude API (Sonnet) / GPT-4o |
| 입력 | 상태 키워드 + 명상 유형 + 소요 시간 + 톤 설정 + 긍정 상태 언어 |
| 출력 | 가이드 스크립트 텍스트 |
| 다양성 | 매번 다른 자연 이미지·표현 사용 |
| 생성 시간 | ~3초 |

프롬프트 구조:

```
"[state] 상태인 사람을 위한 [type] 가이드 스크립트를
[duration]분 분량으로 작성해줘.
톤은 [tone]으로, 마지막엔 [positive_state]를
느낄 수 있도록 마무리해줘.
이전 버전과 다른 자연 이미지와 표현을 써줘."
```

**레이어 2 — 보이스 (TTS AI)**

| 항목 | 내용 |
|---|---|
| 엔진 | ElevenLabs Multilingual v2 / OpenAI TTS |
| 입력 | 스크립트 + 상태별 톤 파라미터 |
| 출력 | 한국어 가이드 음성 (mp3) |
| 생성 시간 | ~10~30초 |

상태별 보이스 톤:

| 상태 계열 | 톤 | 속도 |
|---|---|---|
| 과각성·긴장 | 낮고 안정적 | 0.85x |
| 불안·걱정 | 부드럽고 차분 | 0.90x |
| 무기력·우울 | 따뜻하고 온기 있는 | 0.90x |
| 혼란·과부하 | 명확하고 간결 | 1.00x |
| 회복·전환 | 차분하고 포근 | 0.88x |
| 중립·탐색 | 밝고 자연스러운 | 1.00x |

**레이어 3 — 음악 (AI 음악 생성)**

| 항목 | 내용 |
|---|---|
| 엔진 | Suno API / ElevenLabs Sound Effects |
| 입력 | 유형별 사운드 프로파일 |
| 출력 | BGM + 자연음 (mp3) |
| 생성 시간 | ~30~60초 |

유형별 사운드 프로파일:

| 명상 유형 | 사운드 |
|---|---|
| 호흡 명상 | 무음 또는 바람 소리 |
| 마음챙김 | 자연음 (빗소리·새소리) |
| 바디스캔 | 저주파 싱잉볼·딥 앰비언트 |
| 자비명상 | 따뜻한 피아노·첼로 |
| 시각화 | 자연 앰비언트·물소리 |
| 만트라 | 저주파 드론·옴 사운드 |
| 감사명상 | 밝은 피아노·현악 |
| 수면명상 | 백색소음·빗소리·ASMR |

### 사전 생성 + 캐싱 구조

```
매일 오전 3시 배치 실행
→ 상태 계열 6개 × 유형 8개 = 48개 조합
→ 조합당 3개 버전 생성 = 144개 콘텐츠/일
→ 30일 = 4,320개 누적
→ 사용자는 같은 콘텐츠 반복 경험 없음
```

- 생성된 콘텐츠: AWS S3 저장 → CloudFront CDN 배포
- 진단 완료 시점: 다음 추천 콘텐츠 백그라운드 프리로드
- 배치 스케줄러: AWS Lambda / Cron

### 기술 스택 & 비용 추정 (MVP 기준)

| 항목 | 기술 | 월 비용 추정 |
|---|---|---|
| 스크립트 생성 | Claude API | ~$15/월 |
| 보이스 생성 | ElevenLabs | ~$150~270/월 |
| 음악 생성 | Suno API | ~$8/월 |
| 스토리지·CDN | AWS S3 + CloudFront | ~$5/월 |
| 배치 스케줄러 | AWS Lambda | ~$1/월 |
| 월 총 비용 | | 약 $180~300/월 |

> 콘텐츠 생성 비용은 사용자 수와 무관하게 일정. 한번 생성 후 CDN에서 무한 재사용.

### 경쟁사 대비 콘텐츠 규모
